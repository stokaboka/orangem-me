Описание

Rb разработана для быстрого создания информационных систем с вэб интерфейсом и SQL или NoSQL источниками данных.

Основа Rb - сервер приложений Impress Application Server и SPA вэб приложение реализующее интерфейс пользователя.

В двух словах, разработка вэб приложения для работы с базами с использованием Rb сводится к следующим шагам:

  1. получение схемы базы данных: таблицы и их структура, связи (внешние ключи)
  2. включение в пустую модель выбранных таблиц (источников данных)
  3. применение к выбранным источникам данных связей из схемы базы данных (опционально)
  4. добавить недостающие связи и/или удалить ненужные
  5. настройка свойств источников данных и свойств полей/столбцов: заголовки, фильтры и сортировки по умолчанию, возможность редактирования и сохранения данных и т.д.
  6. настройка  полей/столбцов источников данных  визуальных компонентов для отображения и/или редактирования данных
  7. присвоение модели названия и сохранение.

В результате выполнения этих шагов будет создано вэб приложение позволяющее просматривать и редактировать данные в соответствие с настройками. При изменении выбранной записи выполняется обработка связей. При этом происходит загрузка записей в связанные источники данных.

На данный момент для отображения строк источников данных используется основной UI компонент RbTable. RbTable позволяет просматривать, сортировать и фильтровать записи. В процессе тестирования компонент RbTreeTable

Редактирование записей производится в генерируемой для каждого источника данных форме. Для изменения значений полей/столбцов (выбора значений) и/или фильтрации записей могут использоваться компоненты: RbList, RbComboBox, RbListOfValues, RbSequencer, RbDataMap, RbLinearProgress, RbColorIcon.

Основные элементы Rb

Сервер приложений выполняет функции:

  • является мостом между вэб приложением и различными источниками данных
  • обрабатывает запросы вэб приложения к источникам данных
  • выполняет сервисные операции
  • является вэб сервером для раздачи статических файлов вэб приложения.

Вэб приложение реализует интерфейс пользователя по работе с источниками данных. Структура вэб приложения состоит из нескольких частей:

  • модель;
  • приложение;
  • форма;
  • источник данных;
  • связи между источниками данных.

Модель хранит описание предметной области информационной системы и:

  • определяет источники данных, их структуру и связи;
  • определяет логически завершенные приложения;
  • определяет формы (интерфейс пользователя);
  • определяет связь форм и приложений;
  • определяет связь источников данных

Приложение объединяет формы в логически завершенную часть информационной системы:

  • описание приложения;
  • порядок приложения;
  • коллекция форм;
  • порядок отображения и доступность форм.

Форма является основным элементом интерфейса пользователя. Содержит визуальные компоненты связанные и с источниками данных:

  • описание формы
  • разметка формы
  • коллекция источников данных
  • распределение визуальных компонентов в сетке разметки формы

Источник данных описывает структуру одного объекта базы данных (обычно это таблица):

  • общая информация о базе данных и таблице;
  • возможность чтения/записи;
  • сортировки и фильтры по умолчанию;
  • валидаторы записей;
  • иерархическая структура;
  • столбцы таблицы (поля источника данных).

Столбец таблицы (поле источника данных) определяет тип, хранящейся в источнике (таблице), данных:

  • описание;
  • возможность чтения/записи;
  • сортировки и фильтры по умолчанию;
  • визуальные компоненты для отображения и редактирования данных.

Связи между источниками данных дают возможность выполнять обновление данных и визуального представления в связанных источниках данных при изменении записи в родительском источнике данных.
